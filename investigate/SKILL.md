# 調査スキル

指定された対象について、コードベースを体系的に調査し、レポートを作成する。

## 手順

1. `$ARGUMENTS` をパースし、調査対象を特定する
2. 関連ファイル・コード・設定を網羅的に検索する
3. 以下の観点で分析する:
   - 現在の実装状況
   - 関連するファイルと依存関係
   - 潜在的な問題点やリスク
   - 改善の余地

## 出力フォーマット

```markdown
## 調査レポート: [対象]

### 概要
（1-2文で要約）

### 関連ファイル
- `path/to/file.ts` — 役割の説明

### 現状
（箇条書きで現在の実装状況）

### 問題点・リスク
（あれば箇条書き。なければ「特になし」）

### 推奨アクション
（次にやるべきことを優先度順に）
```

## ルール
- 推測せず、コードに基づいて報告する
- 「〜かもしれません」ではなく「〜です」と断定する。不確実な場合は「未確認: 〜」と明記する
- 日本語で出力する
